# Урок 1.9: Валидация пропсов и React DevTools

Сегодня мы не будем писать сложную логику, а научимся **инструментам отладки**. Это критически важно перед тем, как мы перейдем к состоянию (`useState`) в следующем уроке,.

### 1. Теория

**React DevTools**
Это расширение для браузера (Chrome, Firefox), которое добавляет вкладки `Components` и `Profiler` в инструменты разработчика.

- **Вкладка Components:** Показывает дерево компонентов React (а не просто HTML, как обычный инспектор). Здесь можно увидеть вложенность, какие пропсы переданы компоненту и даже изменять их "на лету" для теста,.
- **Поиск и фильтры:** Если компонентов много, их можно искать по имени или использовать регулярные выражения (например, найти все, начинающиеся на `Header`).
- **Source code:** Можно перейти к исходному коду компонента, нажав кнопку `< >` в панели.

**PropTypes (Валидация пропсов)**
Раньше для проверки типов данных (например, чтобы убедиться, что `age` — это число, а не строка) использовалась библиотека `prop-types`.

- В современном React (с 2024 года и при использовании TypeScript) эта библиотека считается устаревшей и может даже вызывать ошибки линтинга в Vite.
- Однако знать о ней нужно, так как она часто встречается в старом коде (Legacy). Если вы видите ошибку `Failed prop type`, это связано именно с ней.

**Инструкция по проверке (Навигация):**

1.  Убедись, что расширение **React Developer Tools** установлено в твоем браузере.

3.  Нажми `F12` (Developer Tools) и найди вкладку **Components** (она может быть спрятана под значком `>>`).
